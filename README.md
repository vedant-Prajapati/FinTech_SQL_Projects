# FinTech Transactions SQL Analysis – Project Overview

## 📌 Introduction

This project focuses on analyzing **FinTech digital payment transaction data using SQL**.
The repository contains multiple datasets and hundreds of SQL queries designed to simulate **real-world financial transaction analysis scenarios**.

The purpose of this project is to demonstrate how SQL can be used to:

* Explore transaction datasets
* Perform financial analytics
* Identify user spending patterns
* Analyze payment methods and merchant categories
* Practice SQL from **basic to advanced level**

---

# 🎯 Project Objectives

The main objectives of this project are:

1. Understand **FinTech transaction data structure**
2. Perform **data exploration using SQL**
3. Apply **aggregation techniques** to analyze financial patterns
4. Use **subqueries and filtering techniques**
5. Identify **high-value users and transaction trends**
6. Build a **strong SQL portfolio project**

# 📈 Types of Analysis Performed

This project performs various types of transaction analysis:

### Transaction Analysis

* Total number of transactions
* Average transaction amount
* Maximum and minimum transactions

### Customer Analysis

* Total spending per user
* Most active users
* Users with highest transaction amounts

### City-Based Analysis

* Transactions by city
* Total spending by city
* Cities with highest financial activity

### Payment Method Analysis

* Most used payment methods
* Average transaction value per payment method

### Merchant Category Analysis

* Revenue by merchant category
* Category-wise transaction counts
